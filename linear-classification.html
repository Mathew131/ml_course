1.2 Линейная классификация

классификация перцептрон ~12 минут

Теперь предсказываем не число, а класс. Линейный классификатор проводит гиперплоскость (в 2D — прямую) и относит точку к классу по тому, с какой стороны она оказалась: 

ŷ = sign(w·x + b)   →   +1, если w·x + b ≥ 0, иначе −1

Самый ранний алгоритм обучения такой модели — перцептрон Розенблатта (1958). Идея до гениальности проста: пока есть ошибки, бери ошибочно классифицированную точку и чуть-чуть поворачивай прямую в её сторону.

Правило обучения

# для каждой ошибочно классифицированной точки (x, y), y ∈ {−1,+1}:
w ← w + lr · y · x
b ← b + lr · y

Если данные линейно разделимы, перцептрон гарантированно найдёт разделяющую прямую за конечное число шагов (теорема Новикова). Если же классы перемешаны — алгоритм будет «дёргать» прямую вечно: одной прямой их не разделить. Эту проблему позже решат вероятностный подход, максимальный зазор и нелинейные сети.

Интерактив: перцептрон ищет границу

Нажмите Запустить — прямая начнёт поворачиваться, исправляя ошибки. Ошибочные точки в каждый момент обведены кольцом. Можно дорисовать точки кистью выбранного класса: добавьте «перемешанные» точки в чужую область и увидите, как алгоритм перестаёт сходиться.

Что важно унести

  1. Линейный классификатор — это разделяющая гиперплоскость w·x + b = 0.
  2. Перцептрон обучается, подталкивая прямую к ошибочным точкам.
  3. Он сходится только для линейно разделимых данных — отсюда вырастут логистическая регрессия и SVM.
